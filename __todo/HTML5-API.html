Media API The media API is part of the media element which includes two HTML5 children, the video and audio elements. There are a number of Javasript API methods available methods, the play() and pause() as well as load() and canPlayType(). To find out more, take look at the following article HTML5 audio and video: what you must know, NetTuts !! Drag'n'Drop The drag and drop API currently supported in all modern browsers. And it's bring native drag and drop support to the browser. By adding draggable = "true" attribute, the user has ability to move any element. Then add event handlers on a target drop zone to tell the browser where the element can be dropped. Using the API, user could drag and image from the desktop into the browser and more example you may find atHTML5 rocks !! Off line Web Applications . Application Cashe Application caching is carried out by a simple manifest file which lists the files that are required for the application to work off-line. Authors can then ensure their sites functionary off-line. The manifest causes the user's browser to keep a copy of files for use off-line later. When a user view the document / application without network access, the browser swithces to use local copies instead. So in theory you should be able to finish writing an important email or playing web version of Angry birds without connectivity. Read more, on A Beginner's Guide to Using the Application Cache and Using the application cache !! User Interaction Setting the attribute contenteditable = "true" indicates that the element is editable. Using HTML5 History API, developers have a lot more control over history state of user's browser session. Canvas 2D Context Using canvas with 2D Context API allow to draw natively in the browser. The API is extensive and pretty much all canvas articles introduce method and events to work with Integrating Canvas into your Web App and Full Canvas tutorial !! Web Storage When web developers think of storing anything about the user, they immediately think of uploading to the server. HTML5 changes that, as there are now several technologies allowing the app to save data on the client device. There are two main reasons to do this: (a) to make the web app available off-line; (b) to improve performance. Related APIs: Web Storage, Web SQL Database, Indexed Database, and File Access. Awesome article Client-Side Storage and related "Offline": What does it mean and why should I care? !! Web Workers An API that running Javascript in the background independent of any user scripts. Allows for long running tasks without preventing the page from becoming unresponsive. More in The Basics of Web Workers !! Web Sockets Bi-directional communication over the Web: Both the server and client can send data at any time, or even at the same time. Only the data itself is sent, without the overhead of HTTP headers, dramatically reducing bandwidth. Here is a demo !! Server Sent Events The idea behind SSEs may be familiar, a web app "subscribe" to a stream of updates generated by a server and, whenever a new event occurs, a notification is sent to the client. Great article Stream Updates with Server-Sent Events !!
